Add a `@media (prefers-color-scheme: dark)` block immediately after the existing `:root` block (after line 20) in `src/app/globals.css`. The block must override every CSS custom property defined in `:root` with dark-appropriate values.

Insert this exact CSS:

```css
@media (prefers-color-scheme: dark) {
  :root {
    --bg: #0a0a0a;
    --bg-alt: #141414;
    --surface: #1a1a1a;
    --surface-hover: #262626;
    --border: #2e2e2e;
    --border-hover: #404040;
    --text-primary: #f5f5f5;
    --text-secondary: #d4d4d4;
    --text-muted: #a3a3a3;
    --accent: #60a5fa;
    --accent-hover: #93bbfd;
    --accent-light: rgba(96, 165, 250, 0.15);
    --success: #4ade80;
    --warning: #fbbf24;
    --neutral: #737373;
  }
}
```

Color rationale:
- `--bg` #0a0a0a: Near-black background (not pure black, easier on eyes)
- `--text-primary` #f5f5f5: Off-white text for high contrast against #0a0a0a (contrast ratio ~19:1)
- `--text-secondary` #d4d4d4: Light gray for body text (contrast ratio ~13:1 against #0a0a0a)
- `--text-muted` #a3a3a3: Medium gray for muted labels (contrast ratio ~8:1 against #0a0a0a, passes WCAG AA)
- `--accent` #60a5fa: Brighter blue than light mode (Tailwind blue-400, readable on dark bg)
- `--accent-hover` #93bbfd: Even brighter blue for hover states
- `--surface` #1a1a1a: Slightly elevated card/panel background
- `--border` #2e2e2e: Subtle dark borders visible against #0a0a0a and #1a1a1a

Do NOT modify any existing CSS rules, selectors, or the light-mode `:root` block. Only add the new media query block.

After adding the block, scan the rest of globals.css for any hardcoded light-mode color values (hex codes like #171717, #fafafa, etc.) used directly in selectors outside of `:root`. If any exist, they should also be overridden inside the dark media query or converted to use the CSS variables instead. For example, if a selector uses `color: #171717` directly, change it to `color: var(--text-primary)`.
